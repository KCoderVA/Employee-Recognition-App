Supported Award Types

🏥 HRO Awards (High Reliability Organization)

Purpose: Patient safety & quality care

Key Features: Context-Actions-Results format, HRO Principles alignment

Triage → Committee Review → Approval

🦸 Hines Hero Awards

Purpose: Exceptional service above and beyond

Key Features: Detailed impact assessment, service line identification

Submission → Review → Recognition

⭐ Starfish Awards

Purpose: Peer-to-peer recognition for daily excellence

Key Features: I-CARE principles focus, safety flagging

Quick Submission → Immediate Recognition

🌟 ICARE Awards

Purpose: VA core values demonstration

Key Features: Values-based evaluation

Values Assessment → Recognition

Example: Starfish Award Process

User Flow: User Opens App → Selects "Starfish Award" → Fills Form: ├── Nominee Information (auto-populated when possible) ├── Event Date & Location ├── I-CARE Principle Comments: │ ├── Integrity │ ├── Commitment │ ├── Advocacy │ ├── Respect │ └── Excellence ├── Special Flags (Safety/Nursing) └── Additional Comments ↓ Submit → Confirmation → Background Processing: ├── Save to SharePoint ├── Notify supervisor └── Update recognition database

Form Sections Detail

Required Information

  • Nominee name and department
  • Event date and location
  • At least one I-CARE comment

Smart Features

  • Auto-complete from directory
  • Validation prevents errors
  • Save draft capability

HRO Award Approval Workflow

Complex Approval Process: Submission → Triage Team Review → Committee Routing ↓ ↓ ↓ Save to SP → Teams Notification → Approval Action ↓ ↓ ↓ Auto-email → Review Packet → Route OR Return ↓ ↓ ↓ Tracking → Decision Made → Final Processing

Approval Options

✅ Approve & Route

Send to appropriate sub-committee for specialized review

❌ Return for Info

Request additional information from submitter

ℹ️ Add Comments

Include notes and guidance for next reviewer

🔄 Reassign

Route to different committee or reviewer

📱
Reviewers can take action directly in Microsoft Teams or via email
